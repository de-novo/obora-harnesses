# Polysona Next Dry-Run Plan

`polysona` 실제 repo를 대상으로,
Generic Micro-Step System의 read-only dry-run applicability를 검증한다.

## 단계

### Step 1. repo discovery
- tree summary
- markdown inventory
- hooks/scripts inventory
- client/server presence 확인

### Step 2. scoped goal selection
초기 goal은 low-risk 문서 계열로 제한한다.

추천:
- onboarding/readme improvement
- skills documentation consistency review
- persona/content structure explanation doc generation

### Step 3. change-engine simulation
- mixed mode classification
- phase planning
- micro-units derivation
- execution batch derivation

### Step 4. concretization simulation
- broad modify → review-ready concrete shortlist
- docs/skills/personas 중심

### Step 5. execution posture classification
- safe create
- review-only modify
- hold zones
